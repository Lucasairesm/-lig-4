Especificación de Requisitos Conecta 4

REQUISITOS FUNCIONALES

RF-001: Presentación del Tablero de Juego

Descripción: El sistema debe mostrar el tablero a los jugadores, que debe tener 7 columnas y 6 filas.

Prioridad: Esencial.

Asociación:

  • [RF-002] Cantidad de jugadores. [NF-001] La interfaz debe ser intuitiva y fácil de usar.

RF-002: Cantidad de Jugadores

Descripción: Cada partida debe tener exactamente 2 jugadores.

Prioridad: Esencial.

Asociación:

  • [RF-001] Presentación del tablero de juego.

RF-003: Diferenciación entre los Jugadores

Descripción: Cada jugador tendrá un color específico para sus piezas en el juego.

Prioridad: Esencial.

Asociación:

  • [RF-001] Presentación del tablero de juego. [RF-002] Cantidad de jugadores.

RF-004: Control de Piezas

Descripción: Cada jugador juega con 21 piezas y solo tendrá control sobre las piezas de su color elegido.

Prioridad: Esencial.

Asociación:

  • [RF-005] Posicionamiento de las piezas. [NF-001] Interfaz de Usuario. [NF-003] Tiempo de Respuesta. [NF-004] Accesibilidad.

RF-005: Posicionamiento de las Piezas

Descripción: El objetivo del juego es conectar 4 piezas, ya sea en posiciones horizontales, verticales o diagonales.

Prioridad: Esencial.

Asociación:

  • [RF-001] Presentación del tablero de juego. [NF-002] Retroalimentación visual. [NF-003] Tiempo de Respuesta.

RF-006: Identificador de Victoria

Descripción: Después de que un jugador conecte 4 piezas, el sistema identificará y declarará al ganador por el color de las piezas.

Prioridad: Esencial.

Asociación:

  • [RF-003] Diferenciación entre los jugadores. [NF-002] Retroalimentación Visual.

RF-007: Fin del Juego

Descripción: Al declarar al ganador, el sistema debe detener el juego y mostrar una caja de confirmación para que los jugadores puedan reiniciar otra partida.

Prioridad: Esencial.

Asociación:

  • [RF-002] Cantidad de jugadores. [NF-001] La interfaz debe ser intuitiva y fácil de usar.

RF-008: Inicio y Reinicio del Juego

Descripción: El sistema debe sortear qué jugador comenzará o reiniciará el juego.

Prioridad: Esencial.

Asociación:

  • [RF-002] Cantidad de jugadores. [NF-001] La interfaz debe ser intuitiva y fácil de usar.

RF-009: 5 opciones de colores/figuras

Descripción: El sistema deberá ofrecer una selección de 5 colores/figuras para que el jugador elija para sus piezas.

Prioridad: Esencial.

Asociación:

  • [RF-002] Cantidad de jugadores. [NF-001] La interfaz debe ser intuitiva y fácil de usar.

RF-010: Opción de 3 Idiomas

Descripción: El sistema debe permitir al jugador seleccionar hasta 3 idiomas.

Prioridad: Esencial.

Asociación:

  • [RF-002] Cantidad de jugadores. [NF-001] La interfaz debe ser intuitiva y fácil de usar.

RF-011: Interacción Jugador vs. Computadora

Descripción: El sistema debe tener la opción de que el jugador también pueda jugar contra la máquina.

Prioridad: Esencial.

Asociación:

  • [RF-002] Cantidad de jugadores. [NF-001] La interfaz debe ser intuitiva y fácil de usar.

REQUISITOS NO FUNCIONALES

NF-001: Interfaz de Usuario

Descripción: La interfaz debe ser intuitiva y fácil de usar, con instrucciones claras para los nuevos jugadores sobre las reglas y el funcionamiento del juego.

Prioridad: Importante.

Asociación:

  • [RF-001] Presentación del tablero de juego. [NF-002] La retroalimentación visual debe incluir animaciones y efectos gráficos, como visualización de la victoria.

NF-002: Retroalimentación Visual

Descripción: El sistema debe proporcionar retroalimentación accesible y efectiva para las acciones y notificaciones sobre las jugadas, el estado del juego, victorias o empates.

Prioridad: Esencial.

Asociación:

  • [RF-006] Identificador de victoria. [RF-007] Fin del juego. [RF-008] Reinicio del juego. [NF-009] Empate.

NF-003: Tiempo de Respuesta

Descripción: El sistema debe garantizar que las acciones de los jugadores se procesen y se reflejen en la pantalla en menos de 2 segundos.

Prioridad: Esencial.

Asociación:

  • [RF-001] Presentación del tablero de juego: el tiempo de respuesta está directamente relacionado con la rapidez con la que se muestra el tablero al jugador.

NF-004: Accesibilidad

Descripción: El sistema debe permitir la interacción con el uso del mouse.

Prioridad: Esencial.

Asociación:

  • [RF-001] Presentación del tablero de juego.

NF-005: Identificación y Anuncio de Victoria

Descripción: El sistema debe anunciar la victoria de forma visual, proporcionando información clara sobre el ganador.

Prioridad: Esencial.

Asociación:

  • [RF-006] Identificador de victoria.
